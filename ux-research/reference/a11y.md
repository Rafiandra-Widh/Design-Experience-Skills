# Mode: a11y — WCAG 2.2 AA Accessibility Audit

Audit a Figma design or a live page against WCAG 2.2 level AA. Report only what was actually checked.

## Input handling & honest scope

- **Figma design** → `get_screenshot` + `get_design_context` + `get_variable_defs` (real color values, not eyeballed). Design-stage audits CAN check: contrast, target size, type size, color-only signaling, visible focus/state designs, content structure. They CANNOT check: keyboard order, screen-reader semantics, real focus behavior — list these as "perlu diverifikasi saat implementasi".
- **Live page** → Chrome tools (load `claude-in-chrome`). Additionally check keyboard traversal (Tab through the page), focus visibility, zoom to 200%, and inspect semantics via `javascript_tool` (heading tree, img alt, form labels, ARIA misuse).
- Compute contrast ratios from actual hex values — never estimate from a screenshot's appearance.

## Checklist by category (WCAG 2.2 AA)

**1. Kontras & warna**
- Teks normal ≥ 4.5:1; teks besar (≥ 24px / 18.5px bold) ≥ 3:1 (1.4.3)
- Komponen UI & grafik informatif ≥ 3:1 terhadap latar (1.4.11)
- Warna bukan satu-satunya pembeda status/link/error (1.4.1)

**2. Target & interaksi**
- Target sentuh ≥ 24×24 CSS px minimum (2.5.8); praktik baik mobile ≥ 44×44
- Fungsi drag punya alternatif non-drag (2.5.7)
- Tidak ada konten yang hanya muncul saat hover tanpa alternatif (1.4.13)

**3. Teks & struktur**
- Hierarki heading logis, satu H1 (1.3.1); label form terlihat & terhubung (3.3.2)
- Teks bisa diperbesar 200% tanpa kehilangan konten (1.4.4); line-height ≥ 1.5 disarankan
- Bahasa halaman dideklarasikan (3.1.1) — live only

**4. Keyboard & fokus** — live: semua fungsi via keyboard (2.1.1), tanpa jebakan (2.1.2), urutan logis (2.4.3), fokus terlihat (2.4.7) dan tidak tertutup elemen lain (2.4.11). Design: pastikan state fokus DIDESAIN untuk semua elemen interaktif.

**5. Media & gerak**
- Alt text untuk gambar informatif, alt kosong untuk dekoratif (1.1.1)
- Animasi > 5 detik bisa dijeda (2.2.2); hormati prefers-reduced-motion; tidak ada kilatan > 3×/detik (2.3.1)

**6. Form & error**
- Error teridentifikasi dalam teks, bukan hanya warna (3.3.1) + saran perbaikan (3.3.3)
- Autofill/autocomplete untuk data pribadi (1.3.5); login tanpa tes kognitif (3.3.8)

## Output format

```
# Audit Aksesibilitas: [target] — WCAG 2.2 AA
**Cakupan**: apa yang diaudit & metode | **Di luar cakupan**: yang butuh verifikasi implementasi

## Temuan
| # | Lokasi | Kriteria WCAG | Masalah | Nilai aktual | Perbaikan |
(sorted: fail-AA dulu, lalu best-practice warnings — pisahkan keduanya dengan jelas)

## Lolos (kategori yang dicek dan bersih)
## Rekomendasi verifikasi lanjutan (screen reader, pengguna nyata)
```

Every failure cites its success criterion number and the measured value (e.g., "kontras 3.2:1, butuh 4.5:1"). Passing categories are listed too — an audit that only lists failures can't prove coverage.
